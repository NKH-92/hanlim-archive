# 권한 운영 가이드

Admin은 하위 호환을 위해 모든 권한을 가진다. 일반 User는 조회만 가능하며 필요한 권한을 여러 개 조합할 수 있다. 메뉴 숨김은 편의 기능일 뿐이며 모든 쓰기·관리 라우트는 서버에서 권한을 다시 검사한다.

| 권한 | 허용 업무 |
|---|---|
| `can_manage_documents` | 문서 등록·정보 수정, CSV 가져오기, 데이터 품질 수정 |
| `can_move_documents` | 문서 위치 이동 |
| `can_manage_disposals` | 폐기 캠페인 생성·동결·처리·복구 |
| `can_manage_sets` | 세트 생성·수정·잠금·문서 추가/제외 |
| `can_manage_masters` | 랙·대분류·태그 관리 |
| `can_manage_users` | 가입 승인·사용중지·권한 변경 |
| `can_view_audit` | 전역 감사와 위치 이동 이력 조회 |

권장 프로필:

- 조회 사용자: 권한 없음
- 문서고 담당자: 문서관리 + 위치이동 + 세트관리
- 폐기 담당자: 폐기관리
- 운영 관리자: 기준정보관리 + 감사조회
- 시스템 관리자: Admin

권한 변경은 전역 감사로그에 남는다. 사용중지된 사용자는 기존 세션도 다음 요청부터 거부한다. 퇴사·업무변경 시 계정을 삭제하지 말고 먼저 사용중지해 과거 감사 스냅샷을 보존한다. 분기마다 승인 사용자와 권한을 검토하고 최소권한을 유지한다.
