# 개선 백로그

현재 남아 있는 개선 후보만 담는다. 완료·폐기된 항목은 git 이력으로 충분하므로 여기서 지운다.
구조·불변식은 [ARCHITECTURE.md](./ARCHITECTURE.md) 참고.

## 전제

- 이 시스템의 목적은 **문서가 문서고 어디에 있는지 찾는 것**이다. 검색과 위치확인이 핵심이며, 나머지는 이를 보조한다.
- **무료티어 유지**가 운영 조건이다 (Cloudflare Workers 요청 10만/일, D1 5GB — 10인 규모에서는 여유가 크다).
- 소수 인원이 관리하므로 운영 부담이 늘어나는 기능은 보류하고, 감사 대응력과 데이터 신뢰성을 우선한다.
- 판단 기준: 기능 추가보다 **감사 시나리오에서의 소요 시간 단축**.

## 단기 후보 (각 반나절 내외)

1. **세트 인쇄물 실무 양식화**: 인쇄 시 체크 열(□ 확인), 출력 일시, 확인자 서명란 추가 → 종이 한 장으로 확인 대장 역할.
2. **세트 CSV 내보내기**: 세트의 위치 목록을 엑셀로 제출·공유.
3. **미등록 번호 후속 처리**: 세트 일괄 추가에서 "찾지 못한 번호" 클릭 시 해당 번호가 채워진 문서 등록 화면으로 이동.

## 문서고 증설 확정 시

랙은 지금도 관리자 화면에서 추가 가능(구역당 최대 15개). 아래는 **구역 자체가 늘어날 때** 착수한다.

1. 구역 동적 관리: `config.js`의 고정 구역(1~3) 제거, 구역 CRUD 화면 (`floor_plan_regions` 연동).
2. 구역당 랙 상한(15)을 설정값으로 전환.
3. 도면 영역 편집 UI: 현재 `floor_plan_regions`는 SQL로만 수정 가능.
4. 도면 이미지 교체 절차 문서화 (정적 자산이라 재배포 필요).

## 보안·운영 (상시)

1. **계정 사전 등록 화면**: 관리자가 사내 이메일을 먼저 등록하고 임시 비밀번호를 발급하는 흐름. 등록 계정은 최초 로그인 시 비밀번호 변경을 강제한다. 현재는 공개 가입을 닫고 최초 관리자 계정만 migration으로 등록한다.
2. **폰트·아이콘 self-host**: 외부 CDN(jsdelivr/cdnjs) 의존 제거. nonce 기반 CSP는 이미 적용돼 있어,
   self-host 전환 시 `security.js`의 CDN 허용 목록만 지우면 된다.
3. 무료티어 사용량 점검 방법 문서화 (Cloudflare 대시보드 지표 확인 주기).
