# 폐기 캠페인 운영 절차

폐기 캠페인은 전자결재가 아니라 승인된 외부 결재 문서를 근거로 대상 스냅샷을 동결하고 분할 처리하는 도구다. 반출·반납, 실사·검증 상태를 만들지 않는다. 문서별 불변 감사이력은 유지하되 운영 화면에서는 한 번의 정기폐기를 하나의 캠페인 이력으로 조회한다.

1. 폐기 담당자가 문서 폐기 화면 상단의 `정기폐기`를 연다.
2. 폐기 예정 연도와 대분류 중 하나 이상을 선택한다. 서버가 조건에 맞는 현재 보관중 문서의 정확한 총수를 계산하고 결과 전체를 선택한다. 화면은 검토용 앞 100건만 표시할 수 있지만 선택 범위는 필터 결과 전체다. 한 캠페인의 안전 상한은 5,000건이며, 이를 넘으면 연도 또는 대분류 조건을 더 좁힌다.
3. 캠페인 제목, 폐기 사유와 외부 결재 참조를 입력한다. 최종 팝업에 표시된 총 폐기 문서 수를 확인해야 실행할 수 있다.
4. 최종 확인 시 대상 문서 ID, 문서번호, `updated_at`, `row_version`, 위치와 상태 스냅샷을 동결하고 캠페인을 시작한다. 확인 시점의 실제 대상 수가 달라졌으면 아무 문서도 폐기하지 않고 다시 확인하게 한다.
5. 브라우저는 전체 캠페인을 자동 실행한다. 서버는 안전성과 재개 가능성을 위해 내부적으로 25건씩 처리한다. 네트워크가 끊기면 같은 캠페인의 `전체 폐기 재개`로 이어서 처리한다.
6. 동결 후 문서가 수정·이동·폐기되면 해당 항목은 `changed` 또는 이미 처리된 상태로 남기고 자동 폐기하지 않는다.
7. 완료 후 캠페인 이력에서 조건, 사유, 승인 참조, 총 대상, 완료·제외·변경·실패 집계를 확인하고 CSV와 대조한다. 문서별 폐기 이력에서는 캠페인 번호로 원래 캠페인에 이동할 수 있다.
8. 잘못 폐기한 문서는 권한자가 문서 상세에서 사유를 입력해 복구하며, 원래 캠페인 항목과 문서별 감사이력은 삭제하지 않는다.

오류가 반복되면 새 캠페인을 만들지 말고 현재 캠페인의 실패 항목과 Worker 로그 request ID를 확인한다.
