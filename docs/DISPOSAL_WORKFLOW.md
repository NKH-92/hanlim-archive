# 폐기 캠페인 운영 절차

폐기 캠페인은 전자결재가 아니라 승인된 외부 결재 문서를 근거로 대상 스냅샷을 동결하고 분할 처리하는 도구다. 반출·반납, 실사·검증 상태를 만들지 않는다.

1. 폐기 담당자가 조건, 제목, 사유와 외부 결재 참조를 입력해 초안을 만든다.
2. 미리보기에서 대상 수와 위치를 확인한다. 조건이 없거나 0건이거나 상한을 넘으면 동결하지 않는다.
3. 동결하면 대상 문서 ID, 문서번호, `updated_at`, `row_version`, 위치와 상태 스냅샷이 고정된다. 이후 조건은 수정하지 않는다.
4. 필요하면 처리 전에 개별 항목을 사유와 함께 제외한다.
5. 실행은 한 요청에 전체를 처리하지 않고 화면의 다음 처리 버튼으로 분할한다. 네트워크가 끊기면 같은 캠페인에서 재개한다.
6. 동결 후 문서가 수정·이동·폐기되면 해당 항목은 `changed` 또는 이미 처리된 상태로 남기고 자동 폐기하지 않는다.
7. 완료 후 화면·CSV 집계, 문서 감사로그와 폐기로그를 대조한다. 외부 결재번호와 결과 파일을 회사 보존정책에 따라 관리한다.
8. 잘못 폐기한 문서는 권한자가 문서 상세에서 사유를 입력해 복구하며, 원래 캠페인 항목과 감사이력은 삭제하지 않는다.

오류가 반복되면 새 캠페인을 만들지 말고 현재 캠페인의 실패 항목과 Worker 로그 request ID를 확인한다.
