# 무료티어 기반 개선 구현계획

이 문서는 2026-07-17 전달된 실행 명세를 저장소 기준으로 요약한 추적 문서다. 목표는 기능 수를
늘리는 것이 아니라 문서를 찾고, 위치를 바꾸고, 폐기하는 업무를 무료티어 안에서 더 안전하고
짧게 만드는 것이다. 상세 구현 판단은 `ARCHITECTURE.md`, UI 값은 `DESIGN.md`를 우선한다.

## 범위

- 포함: 통합검색, 문서 등록·수정, 통제된 위치 이동과 이력, 7×6 랙 탐색, 폐기 캠페인,
  분할 폐기·재개, 재개형 CSV 가져오기, 세트 CSV·인쇄·잠금, 세분 권한, 전역 감사,
  SQL 페이지 처리, 암호화 백업과 운영 문서, 외부 CDN 제거.
- 제외: 반출·반납, 대여자·가용상태, 위치 확인·검증·실사, 스캔·PDF·첨부파일, R2,
  전자서명·결재, 이메일·메신저 알림, 외부 검색/AI, 실시간 편집, 프레임워크 재작성.

## 내부 예산

`src/freeTierBudget.js`의 값만 사용한다.

- 요청당 D1 statement 목표 40 이하
- 기존 소량 폐기 10건
- 폐기 캠페인 200건, 처리 요청당 25건
- CSV 가져오기 50행, 처리 요청당 1행
- 일반 목록 페이지 30행
- 검색 인덱스 4,000건 경고, 5,000건 재검토

## 구현 단계

1. 기준상태·예산·statement 테스트 기반
2. 기존 일괄폐기 보호, SQL 페이지 처리, 동적 위치 복사, 권한 메뉴, 지원정보 설정화
3. 세분 권한, disabled 사용자, 전역 감사로그와 조회 화면
4. 폐기 캠페인: 초안 → 미리보기 → 동결 → 제외 → 분할 처리 → 보고서
5. CSV 가져오기 작업: 전체 사전검증 → staging → 행별 처리 → 실패 확인·재개
6. 위치 이동 전용 화면·사유·이전/이후 스냅샷·낙관적 잠금
7. 랙 7×6 격자와 rack/face/column/shelf 필터
8. 세트 CSV·인쇄·잠금, 데이터 품질 작업목록, 검색 규모 경고
9. 외부 CDN 제거, gzip+AES-256+SHA-256 백업, 복구·권한·폐기·무료티어 runbook

## 핵심 데이터 규율

- 폐기 대상은 조건을 다시 실행하지 않고 동결 당시 문서·위치·`updated_at`·`row_version`을 스냅샷으로 저장한다.
- 폐기는 processing token으로 항목을 선점하고 집합 SQL 한 batch로 처리한다.
- 동결 뒤 바뀐 문서는 `changed`로 남기고 자동 폐기하지 않는다.
- CSV와 폐기는 화면을 닫아도 완료된 상태가 D1에 남고 사용자가 수동으로 재개한다.
- 위치 이동은 문서 감사 → 이동 이력 → 문서 UPDATE 순서의 단일 batch로 처리한다.
- 과거 감사·폐기 보고서는 사용자나 문서가 비활성화·삭제돼도 보존한다.

## 완료 조건

- 검색 없는 목록이 SQL `COUNT + LIMIT/OFFSET`으로 동작한다.
- 동결 목록 밖 또는 동결 뒤 변경된 문서는 캠페인에서 폐기되지 않는다.
- 폐기와 CSV 작업은 중단 후 재개되고 중복 로그·중복 문서가 없다.
- 위치 이동 전후와 사유가 보존되며 동시수정은 거부된다.
- 랙 셀 건수와 위치 필터 목록이 일치하고 세트 CSV·인쇄·잠금이 동작한다.
- 권한 없는 직접 URL은 403이며 disabled 사용자의 기존 세션은 즉시 무효다.
- 외부 CDN 없이 화면이 동작하고 암호화 백업·복구 절차가 문서화된다.
- 변경마다 `npm run check && npm test`를 통과한다.
