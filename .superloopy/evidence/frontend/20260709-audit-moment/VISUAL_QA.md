# 시각 QA — 테마 2 "감사 순간" (붙여넣기 픽리스트 · 감사 응답 카드)

날짜: 2026-07-09 · 검증: `wrangler dev` 로컬 + 실 D1, curl 서버렌더(인라인 script 제거 후 판정) + 브라우저

## 1) 붙여넣기 → 동선순 픽리스트 (`/picklist`)
- `GET /picklist` 200 (붙여넣기 폼). 진입점: 세트 페이지 헤더 "붙여넣기 픽리스트" + 커맨드 팔레트.
- `POST /picklist` with `NB-2026-100 NB-2026-101 NB-2026-103 NOPE-999 ARC-000001`:
  - 요약: **"요청 5건 · 찾음 4건 · 못 찾음 1건"** · 4개 랙.
  - 못 찾은 번호 배너(danger 좌측 레일): 칩 `NOPE-999` 하나만 — 걷기 전 해명 가능.
  - 테이블 동선순(구역→랙→면→열→선반): `MR-2026-001`(=ARC-000001 storage_code 매칭), `NB-2026-100/101/103`.
  - 반출 중 행(NB-2026-103) `is-checked-out` 노랑 레일.
  - 체크오프(`data-picklist data-pick-set="adhoc"`): 브라우저에서 첫 행 체크 → `is-picked` + 회수 카운트 "1". 콘솔 오류 0.
- 버그 수정: `parseDocumentNumberList`가 공백 미분리 → "NOPE-999  ARC-000001"이 한 토큰. 파서를 `/[\s,;]+/`로 고쳐 공백 구분 지원(세트 일괄추가에도 이득), 단위 테스트 추가.

## 2) 감사 응답 카드 (`/documents/:id/custody`)
- 진입점: 문서 상세 위치 패널 "감사 응답" 버튼. 모든 인증 사용자 접근(감사관에게 제시용).
- 반출 중 문서(NB-2026-103) 카드 렌더(계산 텍스트 = UTF-8 정상, 아래는 실제 값):
  - 현재 상태: 보관중 + 반출 중 · 김검사
  - 소재: **"반출 중 · 김검사님 보관"**
  - 보관 위치: `1구역 / 2번 랙 / 1열 / 1선반 / A면`, 보관코드 `ARC-000022`, 대분류 제조기록서
  - 반출 정보: 김검사 · 2026-07-09 14:28:47 · 감사
  - 최근 이동: QA Tester · 2026-07-09 14:16:30
  - 소재 이력(반출+이동 시간순) 2건, 인쇄 버튼(`data-print`).
  - 브라우저: `.custody-card` 렌더, 콘솔 오류 0.
